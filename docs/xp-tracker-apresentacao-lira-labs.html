XP Tracker

Plataforma web para acompanhar progresso, níveis, XP restante, histórico e rotas de farm em jogos de RPG via bot no Telegram.

Resumo executivo

O XP Tracker transforma informações dispersas de um jogo por bot em uma dashboard simples, visual e segura. O jogador consegue saber onde está, quanto falta para upar e quais atividades ajudam a chegar mais rápido ao próximo nível.

O produto foi pensado para jogadores individuais, comunidades, guildas e administradores que desejam mais clareza sobre evolução, metas e desempenho.

O problema e a solução

Para que o site serve

Contexto do jogo

Em jogos via bot no Telegram, o jogador geralmente precisa acompanhar XP, nível, progresso, caçadas, criptas e masmorras com informações espalhadas em mensagens, tabelas ou prints.

Isso torna difícil entender quanto falta para subir de nível e qual atividade compensa mais naquele momento.

Solução

O XP Tracker centraliza esse acompanhamento em uma dashboard web, com histórico, estimativas, cálculo de progresso, conquistas e planejamento de runs.

O jogador pode testar como visitante e, se gostar, salvar o progresso na nuvem usando Google.

Fluxo principal

01

Entrar

O usuário entra como visitante ou com Google. No primeiro contato, o nível começa em 0.

02

Configurar

Informa nível atual, nível alvo, XP para upar, XP restante e meta diária.

03

Acompanhar

Visualiza progresso, conquistas, histórico, gráfico e estimativas de conclusão.

04

Salvar

Ao logar com Google, o progresso do visitante é levado para a conta e salvo automaticamente.

Segurança e confiança

Níveis de segurança implementados

Nível 1

Autenticação segura

Implementado: login com Google via Supabase Auth. O usuário visitante não grava dados na nuvem até escolher autenticar.

Nível 2

Isolamento por usuário

Implementado: Row Level Security no Supabase. Cada conta autenticada só pode ler, criar e atualizar o próprio progresso.

Nível 3

Regras de banco

Implementado: políticas de SELECT, INSERT e UPDATE baseadas em auth.uid() = user_id, além de linha única por usuário.

Nível 4

Hospedagem e transporte

Implementado: publicação em Cloudflare Pages com HTTPS e headers de segurança configurados no projeto.

Nível 5

Boas práticas operacionais

Implementado: versionamento no GitHub, deploy controlado e separação entre chave pública do cliente e regras reais no banco.

Pontos de confiança para venda

Privacidade do progresso

O progresso salvo fica associado à conta autenticada. Outro usuário não consegue acessar ou alterar os dados dessa conta via API pública.

Sem bloqueio para testar

O modo visitante reduz atrito comercial: a pessoa experimenta primeiro e só faz login quando percebe valor no produto.

Arquitetura

Como o produto está estruturado

Camada Tecnologia Função no produto
Frontend React Router, React, Tailwind CSS Interface web, dashboard, formulários, gráficos, planejador e experiência responsiva.
Autenticação Supabase Auth + Google OAuth Entrada segura do usuário e sessão autenticada para salvar progresso.
